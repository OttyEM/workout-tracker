# Workout Tracker

## What this is
A personal workout tracker for my own use. Single user (just me),
used mainly on my iPhone in the browser. Not a product — simplicity
beats features.

## Tech & constraints
- Plain HTML, CSS, and JavaScript. No build step, minimal dependencies.
- Data is stored locally in the browser (IndexedDB). Nothing is sent
  to a server. Keep it that way unless I explicitly ask for sync.
- Must work well on a phone screen first.


## Design principles
- Mobile-first, minimal, large tap targets.
- Logging a workout should be fast and require as few taps as possible.

## How I like to work
- I'm rusty at coding, so explain your changes briefly and avoid
  unexplained jargon.
- Make one change at a time so I can check each result.
- Commit to git when something works; ask before any big refactor.
